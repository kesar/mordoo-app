System Flow

Daily Pulse Reminder — End to End
📱 Mobile App
1. Request OS permission
2. Get Expo Push Token
3. Save token to profiles
4. Save time preference
5. Receive & display notification
→
🗄 Supabase
profiles table
+ push_token (text)
+ notifications_enabled (bool)
+ notification_time (time)
+ timezone (text)
pg_cron
Runs every 15 min, finds users
whose notification_time is now
→
⚡ Edge Function
1. Query eligible users
2. Batch push tokens
3. Call Expo Push API
4. Handle failures/receipts
5. Mark as sent

Components to Build

Mobile App

  • expo-notifications setup
  • Permission request in Power-Ups
  • Push token registration service
  • Settings: toggle + time picker
  • Notification handler (tap → Pulse)

Database

  • Migration: add columns to profiles
  • RLS policy for push_token
  • pg_cron job (every 15 min)
  • last_notification_sent tracking

Edge Function

  • send-daily-reminder function
  • Expo Push API integration
  • Batch sending (100 per request)
  • Error handling & token cleanup

API Endpoint

  • POST /api/notifications/register
  • Save/update push token
  • Update notification preferences
